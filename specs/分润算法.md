### 一、查询按日期查询分润的订单数据同时按订单归属的代理商分组汇总分润金额，SQL如下
```sql
SELECT mch.agent_id, SUM(merchant_profit) AS merchant_profit FROM merchant_order AS odr
JOIN merchant AS mch ON odr.merchant_id = mch.id
WHERE mch.agent_id > 0 AND odr.bill_date = ''
GROUP BY mch.agent_id
```

如果没有查询到数据则直接结束任务

### 二、查询参与分润的上级代理商数据，SQL如下：
```sql
SELECT supagent.id,supagent.agent_id,supagent.agent_rate,subagent.`level` 
FROM
(
    SELECT @id AS c_ids, (SELECT @id := agent_id FROM `user` WHERE `id` = @`id`) AS p_ids ,@l := @l + 1 AS `level` 
		FROM `user`, (SELECT @`id` := 最低一级代理商id, @l := 0) b 
) AS subagent
JOIN `user` AS supagent ON subagent.c_ids = supagent.id
```

### 三、分配逻辑

查询代理商的所有上级代理商并逐级进行分配，如果中间某一级的分润比例为0，则不在继续往下分配。
举例：代理商A的上及代理商依次是 AgentB（10%）-->AgentC-->（5%）-->AgentD（0%）-->AgentE（10%），代理商A的商户产生的利润为100元，需要抽取10元给AgentB，代理商A剩余90元，AgentB获得的10元抽成需要再抽取5%给AgentC，因为AgentC给AgentD的比例为0，则后续的AgentD和AgentE不会得到抽成。
