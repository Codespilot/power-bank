# 功能描述

本模块用于管理用户上传的文件

# API接口

上传：POST /api/attachments/upload/form|base64
列表：GET /api/attachments
获取：GET /api/attachments/{id}
删除：DELETE /api/attachments/{id}
预览：GET /files/attachments/{uuid}?token=xxx&thumb=100x100

列表和删除接口不显示在Swagger，获取接口不包含signature_key。

# 功能

## 上传

用户通过form表单上传时调用`/api/attachments/upload/form`，文件相关信息从表单读取
如果通过base64上传则调用`/api/attachments/upload/base64`，body内容格式如下：
```json
{
    "name":"文件名.jpg", // 必传
    "content": "xxxxxx", // 必传
    "extension": ".jpg"
}
```
上传成功响应消息格式：
```json
{
    "file_name": "uuid.jpg",
    "file_size": 1234,
    "file_ext": ".jpg"
}
```
上传失败响应消息：
```json
{
    "message": "错误信息"
}
```
上传时生成随机字符串signature_key用于token签名。

/api/attachments开头的接口均需要认证。

## 预览

访问地址 `/files/attachments/{uuid}?token=xxx&thumb=100x100`
参数：
- uuid：上传接口返回给用户的file_name，对应attachment表的file_name。
- token：后端生成的JWT access_token，记录file_name(token的subject部分)，有效期（120分钟）等，访问是需要校验是否有效
- thumb：图片的宽高，非图片文件忽略此参数，如果是图片文件，需要根据参数值进行缩放，缩放模式为AspectFill
逻辑处理流程：拦截用户请求-->校验token(比对uuid与subject是否相等，token是否有效)-->读取文件-->裁剪
响应的content-type通过config/mime.py映射获取，字典的key为文件扩展名。

## 生成token

提供生成文件访问token的公共方法，入参为file_name，通过file_name查询attachment表对应的记录，file_name作为token的subject，file_ext对应token的ext字段，使用signature_key进行签名。

token可以无限生成，不需要保存
